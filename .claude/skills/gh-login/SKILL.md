## 概要

`gh` コマンドを使う前に認証状態を確認し、未ログインの場合はユーザーにログインを案内する。
PR 作成・Issue 操作・リポジトリ参照など `gh` を必要とする操作の前に自動起動する。

## 起動条件

- `gh pr create` / `gh pr list` / `gh issue` などの操作を行う前
- 「PR を作って」「gh でログインして」などの自然言語
- `/gh-login` 明示起動

## 実行フロー

1. `gh auth status` を実行して認証状態を確認する
2. **認証済みの場合** → 「GitHub CLI は認証済みです（{ユーザー名} / {ホスト}）」と表示して終了
3. **未認証の場合** → 以下を案内する

```
GitHub CLI が未認証です。ターミナルで以下を実行してください：

  ! gh auth login

ブラウザが開くので GitHub アカウントでログインしてください。
完了したら教えてください。
```

4. ユーザーから完了の返答を受けたら再度 `gh auth status` で確認する
5. 認証が確認できたら中断していた操作（PR 作成など）を再開する

## 注意事項

- `gh auth login` はインタラクティブな操作が必要なため Claude が直接実行することはできない
- ユーザーに `! gh auth login` と入力してもらうことで、Claude Code のターミナルセッションに出力が流れる
